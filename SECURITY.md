# Security Policy

## Supported versions

Security fixes are applied to the `main` branch.

If you run older commits, upgrade to latest `main` before reporting a vulnerability unless your report is specifically about upgrade regressions.

## Reporting a vulnerability

Please do **not** open public issues for vulnerabilities.

Report privately to:
- Email: **salvatierra.pablom@gmail.com**
- Subject: `SECURITY: Moltbook Observatory`

Include:
- Affected component/file
- Impact summary
- Reproduction steps or proof of concept
- Suggested remediation (if available)

## Response targets

Best-effort targets:
- Acknowledgement: within 72 hours
- Initial triage: within 7 days
- Fix or mitigation plan: as soon as practical based on severity and reproducibility

## Disclosure policy

- We prefer coordinated disclosure.
- Once fixed, we may publish a summary with impact, affected versions/commits, and remediation steps.

## Scope notes

This repository may process public platform data. Security reports should focus on:
- Code execution or supply-chain risks
- Credential/secrets exposure
- Data handling risks caused by repository code/configuration

Platform-level vulnerabilities unrelated to this repository should be reported to the relevant platform owner.
